# 디지털 자산 통합 관리 전략 제안 (PARA 원칙 기반)

20년 경력의 베테랑이시라도 카테고리 정리에 어려움을 느끼신다는 점에 깊이 공감합니다. 경험이 많을수록 관리할 정보의 양과 종류가 방대해져서 통일된 원칙 없이는 금방 혼란에 빠지기 쉽습니다. 특히 WSL, 윈도우, 옵시디언, 북마크처럼 각기 다른 성격의 시스템을 하나로 묶는 것은 매우 어려운 과제입니다.

PARA 원칙을 대전제로 삼으신 것은 훌륭한 출발점입니다. 이제 문제는 '어떻게' 이 원칙을 각 시스템에 '일관되게' 적용하느냐입니다.

제가 제안하는 핵심 원칙은 **'정보의 성격과 실행 가능성(Actionability)'**에 따라 분류하는 것입니다.

---

## PARA 원칙의 재해석: 모든 것을 꿰뚫는 하나의 질문

어떤 정보(파일, 폴더, 노트, 북마크)를 마주했을 때, 스스로에게 이 질문을 던져보세요.

> **"이것이 '명확한 목표와 끝'이 있는 일과 관련이 있는가?"**

- **YES (네):** **`Projects`** 로 갑니다. (예: 'karakeep 기능 개발', '여름 휴가 계획', '자격증 시험 준비')
- **NO (아니오):** 그렇다면 이것이 **"내가 책임감을 갖고 지속적으로 관리해야 할 영역인가?"**
    - **YES:** **`Areas`** 로 갑니다. (예: '건강', '재정', '커리어 개발', '팀 관리')
    - **NO:** 그렇다면 이것이 **"미래에 유용할 수 있는, 관심 있는 주제인가?"**
        - **YES:** **`Resources`** 로 갑니다. (예: 'AI 에이전트', '파이썬', '요리법', '가드닝')
        - **NO:** 그렇다면 이것은 **"지금 당장은 필요 없는, 완료되거나 보관할 자료인가?"**
            - **YES:** **`Archive`** 로 갑니다. (예: '완료된 프로젝트', '더 이상 관리하지 않는 영역의 자료')

이 질문의 흐름이 모든 시스템을 관통하는 통일된 기준이 될 것입니다.

---

## 시스템별 PARA 적용 가이드

### 1. 파일 시스템 (WSL, Windows 디렉토리)

파일 시스템은 '실행'과 '산출물'이 저장되는 공간입니다. 가장 엄격하게 PARA 구조를 적용해야 합니다.

**구조 제안:**
사용자 홈(`~/` 또는 `C:\Users\YourName`) 아래에 모든 것을 시작할 단 하나의 루트 폴더를 만듭니다. (예: `~/mydocs`, `C:\Users\YourName\DigitalLife`)

```
/home/your_user/docs/  (또는 C:\Users\YourName\Docs)
├── 1_Projects/
│   ├── karakeep-feat-auth/      # 진행 중인 실제 프로젝트 폴더
│   ├── 2026-q3-presentation/    # 명확한 마감일이 있는 업무
│   └── personal-home-server-setup/
│
├── 2_Areas/
│   ├── professional/            # 직업적 책임 영역
│   │   ├── team-management/
│   │   └── career-dev/
│   ├── personal/                # 개인적 책임 영역
│   │   ├── finances/
│   │   └── health/
│   └── karakeep-maintenance/      # 특정 프로덕트의 유지보수처럼 '끝이 없는' 관리 영역
│
├── 3_Resources/
│   ├── topics/                  # 관심 주제별
│   │   ├── ai-agent-dev/
│   │   ├── system-architecture/
│   │   └── investment-strategies/
│   ├── tools/                   # 도구/라이브러리 관련
│   │   ├── python/
│   │   └── fast-api/
│   └── templates/               # 자주 쓰는 문서/코드 템플릿
│
└── 4_Archive/
    ├── 2025-completed-project-alpha/ # 완료된 프로젝트
    └── old-tax-records-2024/         # 보관이 필요한 과거 자료
```

**핵심 팁:**
- 폴더명 앞에 `1_`, `2_` 와 같은 숫자를 붙이면 항상 정렬 순서가 고정됩니다.
- **WSL과 Windows 간의 일관성:** WSL 홈(`~/`)과 Windows 문서 폴더 양쪽에 동일한 `1_Projects`, `2_Areas`... 구조를 만드세요. 중요한 것은 두 시스템이 파일을 공유하는 지점(예: `C:\Users\YourName\docs`를 WSL에서 `/mnt/c/Users/YourName/docs`로 접근)을 정하고, 그곳을 PARA의 중심으로 삼는 것입니다.

### 2. 옵시디언 (Obsidian - 두 번째 뇌)

옵시디언은 파일 저장을 넘어 '생각의 연결'을 위한 공간입니다. 폴더 구조와 태그를 함께 사용합니다.

**구조 제안 (폴더 기반 + 태그 보조):**
옵시디언 Vault(저장소) 루트에도 파일 시스템과 동일한 PARA 폴더를 만듭니다.

```
MyObsidianVault/
├── 10_Projects/
│   ├── [[karakeep-feat-auth MOC]] # 프로젝트의 목차(Map of Content) 노트
│   ├── 2026-06-12-회의록.md
│   └── 아이디어-인증-플로우.md
│
├── 20_Areas/
│   ├── [[Health MOC]]
│   ├── [[Career Development MOC]]
│   └── 2026년-자기계발-계획.md
│
├── 30_Resources/
│   ├── AI 에이전트.md  # 관련 노트들을 링크하는 MOC 역할
│   ├── FastAPI의-존재-이유.md
│   └── SQLModel-vs-SQLAlchemy.md
│
├── 40_Archive/
│   └── (완료된 프로젝트 노트들이 이동)
│
└── 99_Inbox/  # !!중요!! - 일단 모든 생각/메모는 여기에!
```

**핵심 팁:**
- **Inbox의 활용:** 웹클리핑, 갑자기 떠오른 아이디어, 정리되지 않은 메모 등 모든 것은 `99_Inbox` 폴더의 '일일 노트(Daily Note)'에 먼저 기록합니다.
- **주간 리뷰:** 매주 시간을 내어 Inbox의 노트들을 PARA 폴더로 옮기며 정리(Processing)합니다. 이때 위의 핵심 질문을 사용합니다.
- **태그 활용:** 노트 하나는 하나의 폴더에만 존재할 수 있지만, 태그는 여러 개를 가질 수 있습니다.
    - 예: `FastAPI의-존재-이유.md` 파일은 `30_Resources` 폴더에 있지만, `#project/karakeep-feat-auth` 태그를 달아 현재 프로젝트와의 연관성을 표시할 수 있습니다.

### 3. 브라우저 북마크

북마크는 외부 '리소스'를 저장하는 장소입니다. PARA 구조를 단순화하여 적용합니다.

**구조 제안:**
북마크 바에 최상위 폴더 4개만 만듭니다.

```
- 1_Projects/  # 현재 프로젝트 관련 링크 (JIRA, GitHub Repo, Figma 등)
  - karakeep-auth/
    - JIRA-Ticket-123
    - GitHub PR #45
- 2_Areas/     # 자주 방문하는 관리성 사이트 (은행, 사내 시스템, 건강 정보 사이트)
  - Finances/
    - 우리은행 로그인
    - 증권사 MTS
- 3_Resources/ # 기술 문서, 튜토리얼, 블로그 글
  - Python/
    - Python 공식 문서
    - Real Python 블로그
- Inbox/       # !!중요!! - 나중에 읽을 글, 일단 저장
```

**핵심 팁:**
- **'나중에 읽기(Read Later)' 서비스 활용:** Pocket이나 Instapaper 같은 서비스를 사용하거나, 간단히 `Inbox` 폴더를 만들어 일단 모든 링크를 여기에 던져 넣습니다. 북마크가 어지러워지는 주된 원인은 '언젠가 볼 것'들이 `Resources` 폴더를 오염시키기 때문입니다.

### 통합 워크플로우: 모든 것을 하나로

1.  **수집 (Capture):** 모든 정보(파일, 아이디어, 링크)는 각 시스템의 **`Inbox`** 로 들어옵니다.
    - 파일 -> `~/docs/0_Inbox`
    - 아이디어/메모 -> 옵시디언 `99_Inbox`의 Daily Note
    - 웹 링크 -> 북마크 `Inbox` 폴더 또는 Pocket

2.  **정리 (Process):** 매일 또는 매주 정해진 시간에 `Inbox`를 비웁니다. 각 항목에 대해 핵심 질문을 던져 PARA 폴더로 옮깁니다.

3.  **실행 (Execute):** `1_Projects` 와 `2_Areas` 폴더/노트를 보며 오늘의 할 일을 결정하고 실행합니다.

4.  **보관 (Archive):** 프로젝트가 끝나면 `1_Projects`의 관련 폴더/노트 전체를 `4_Archive`로 옮깁니다. 이것이 시스템을 깨끗하게 유지하는 비결입니다.

### 첫 걸음 제안

이 모든 것을 한 번에 바꾸려고 하면 지치기 쉽습니다.

**오늘 당장 `~/` 나 `C:\Users\YourName` 아래에 `1_Projects`, `2_Areas`, `3_Resources`, `4_Archive` 폴더 4개를 만드는 것부터 시작해 보시는 건 어떨까요?**

그리고 다음 일주일간, 새로 생기는 모든 파일만이라도 이 4개 폴더에 분류하는 연습을 해보세요. 점진적으로 기존 파일들을 옮겨가면 부담 없이 새로운 시스템에 정착하실 수 있을 겁니다.
